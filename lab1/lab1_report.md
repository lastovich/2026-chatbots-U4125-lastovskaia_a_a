## Fly me to the Moon 🌕  
### Описание задачи:
Я - преподаватель английского. Мои ученики школьного возраста, и, помимо основной задачи обучения, я хочу обучить их умению рефлексировать и давать обратную связь.  
Так, я выбрала создать бота по планированию занятий, сбора обратной связи и еженедельных отчетов. Бот будет доброжелательным и радовать моих учеников стикером после проведенного занятия.  
### Создание промпта
* Сперва с помощью Grok уточнила функционал своего бота  
  ```
  Бот должен:
  Позволять учителю вручную добавлять учеников
  Принимать расписание занятий на ближайшие 2 дня пошагово
  Через 1 час после запланированного времени автоматически спрашивать у учителя подтверждение, прошёл ли урок
  После подтверждения учителя «урок прошёл»:
  Запрашивать у учителя оценку взаимодействия и усвоения материала
  «Отправлять» ученику стикер-приз (заглушка + подробный лог)
  Сразу после этого «отправлять» ученику опросник обратной связи (заглушка + лог)
  Если ученик не ответил на опрос в течение 30 минут — «отправлять» напоминание (заглушка + лог)
  Вести статистику проведённых уроков, тем и оценок
  Каждую субботу в 19:00 автоматически формировать и присылать учителю сводный еженедельный отчёт
  ```

### Стек технологий (описан с помощью Grok)

| Библиотека              | Почему выбран именно она |
|-------------------------|---------------------------|
| **aiogram** (>=3.7.0)   | Основной асинхронный фреймворк для создания Telegram-ботов. Выбран специально вместо python-telegram-bot, потому что лучше подходит под задачи бота: удобная работа с состояниями (FSM), пошаговые диалоги (ввод расписания), интеграция с планировщиком задач и высокая производительность. Это современный и рекомендуемый выбор для ботов средней и высокой сложности в 2026 году. |
| **APScheduler**         | Мощный и удобный планировщик задач. Используется для автоматических действий: проверка прошедших уроков через 1 час, отправка напоминаний ученикам и еженедельного отчёта каждую субботу в 19:00. |
| **python-dotenv**       | Удобная загрузка переменных окружения из файла `.env` (в первую очередь `BOT_TOKEN`). Позволяет безопасно хранить токен бота вне кода. |

### Дополнительные замечания

- На текущем этапе бот использует **JSON-файл** для хранения всех данных (ученики, уроки, отчёты). Это сделано для максимальной простоты и удобства разработки. SQLight будет подключен на следующих этапах лабораторной работы.  
- Все взаимодействия с учениками пока реализованы как заглушки с выводом в консоль.
- Код написан асинхронно, что позволяет боту корректно работать с таймерами и планировщиком.
